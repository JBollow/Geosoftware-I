Manuelles Hinzufügen von Etappen
	Eine Etappe hat einen Name, einen Start und ein Ziel, Termine (Anfangsdatum, Enddatum), Links zu Bildern, Webseite….
	Hochladen von Fotos ermöglichen (für Start und Ziel)
	Möglichkeit, Bilder und Beschreibungen als Marker ‘entlang der Strecke’ hinzuzufügen
	
Manuelle Digitalisierung von Objekten
	z.B. Parkplätze, Zuschauerplätze
	notwendige Eigenschaften definieren (Name, Preis, Kapazität)
	
Import Funktionalität
	Hochladen von einzelnen Objekten oder ganzen Etappen aus geoJSON-Datei
	Textfeld-Import von GeoJSON Objekten
	Fehlerüberprüfung
	
Navigation
	vom Start/Ziel/Zuschauerplatz zum nächstgelegenenParkplatz sollt ermöglicht werden
	Der Navigations-Pfad zwischen Parkplatz und jeweiligem Streckenabschnitt soll automatisch wieder aus der Karte entfernt werden, wenn der User ein anderes Objekt oder Event “anwählt”
	Minimalanforderung: Auto-Navigation
	multi-modaler Verkehr (z.B. Bus, Fahrrad, zu Fuß)
	
Speichern
	von Objekten und Etappen in eigener MongoDB
	Laden soll wieder möglich sein
	Strecken bearbeiten und ggfs. wieder löschen
	Einrichten von Permalinks zu allen Objekten
	
Suche
	nach Etappen (nach Start, Ziel und Name)
	nach Terminen
	nach Parkplätzen für eine Etappe nach Kapazität.
	
Layout & Design
	Webseite soll eine Startseite und ein Impressum haben
	Für die geforderten Funktionen dürfen mehr als eine Seite erstellt werden
	Karte soll auch auf dem Smartphone gut bedienbar sein
	
Sonstiges
	Sinnvolles Logging gestalten
	Zu jedem Start & Ziel einer Etappe soll automatisch der erste Absatz des dazugehörigen Wikipedia-Artikels eingebunden werden (wenn vorhanden)
	
Komponententest
	npm test
	Mindestens 5 sinnvolle Unit Tests mit Mocha.js
	https://mochajs.org/
